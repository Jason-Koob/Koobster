My Python Timer

I wrote a python script to remind me every second. It's basically a standard timer, but I wanted to make it myself, without much external help. Firstly I needed to get the current time and the desired time. I looked for the only major python library I would need for this project. It is called datetime. I used this library to get the current time and find the difference between desired time and the current time. I quickly realized I needed to get a format of time to convert the current and the standard times to work together through the library. I found out how to do that with a minimal amount of research online.

* Not like doing research online is a bad thing. I just want to make something myself instead of going off a tutorial or else I wouldn't learn anything from it.

So I wrote a function to ask the user for their desired time, asking them to use a year-day-month hour:minute:second format. I then combined their input with a format so the library could comprehend their answer as a time instead of a string of gobbledygook. Now all that was left was to make the actual timer. I used a while loop that re-acquired the current time every second thanks to the time python library I also used. this library was really only used to wait an amount of time before re-acquiring the current time. Then the date time library would subtract the current time from the current time and print the difference.

Blackjack Card Game Emulator

I expected this project to be finished very quickly because I thought I knew how to play the 'simple card game' known as blackjack. I thought all you had to do was get cards with a value of 17 and not 21 and hope you win. I was mostly correct in thinking that, except I forgot how to make things simple when writing code.

I once again used the random python library to randomly choose an item from a list. This time I needed to add the randomly choosen item to another list and assign a value to each item in this second list, then combine the values of this second list for a variety of values. I did this twice, once for the user and again for the dealer. I had to re-learn how to play blackjack. Upon doing so, I revised my code and copied over some of my functions. I then revised my code again in a now third attempt at making Blackjack in python. I told one of my friends about me making Blackjack and he joined in and helped me remember how to play the game. Thanks Jack.

I finally got all of the issues sorted out and got a working budgetting system in place that reads and writes data from a file and allows the user to bet an amount of lesser or equal value from the value in the file. This function took some time but is well worth it. Now it feels like I'm actually losing something when I lose instead of just retrying. This project took me a few days and is well worth the effort put into it.
